How Densa Deck computes everything

Full transparency on every number the app shows you. Each section below maps a UI feature to the Python module that produces it, the inputs it uses, and the formulas it applies. Nothing is hidden; nothing is magic.

Power level (1-10) + sub-scores

Implemented in densa_deck.analysis.power_level.estimate_power_level.

The overall power level is a weighted average of six sub-scores, each on a 0-10 scale. Weights reflect how predictive each axis is for real games, tuned against a calibration set of known-power decks (precons at the low end, cEDH optimized builds at the high end).

overall = sum(sub_score[i] * weight[i]) / sum(weight[i]) weights: speed 1.5 interaction 1.2 combo_potential 1.5 mana_efficiency 1.3 win_condition_quality 1.2 card_quality 1.0

Tier thresholds

After averaging, the score maps to a human-readable tier so Rule 0 conversations don't have to quote decimals:

  • jank — below 3.0
  • casual — 3.0-5.0
  • focused — 5.0-7.0
  • optimized — 7.0-8.5
  • competitive — 8.5-9.5
  • cEDH — 9.5+

Sub-scores

Speed. How fast the deck can threaten lethal or close a game. Derived from mana curve shape (peak at 2-3 is fast, peak at 4+ is slow), ramp piece density, and low-CMC threat count. A fast deck is typically 1-3 turns faster to its primary win than a slow one.

Interaction. Quantity + quality of disruption. Counts targeted removal, counterspells, and board wipes, then compares to format-specific targets (Commander: 8-12 interaction; Modern: 6-10). Tagging is rule-based and lives in densa_deck.classification.tagger.

Combo potential. Tutor density + reliable wincon engine density + low-CMC win conditions. High when the deck can reliably assemble a win that bypasses combat.

Mana efficiency. How consistently the deck can cast what it draws. Uses land count against format target, ramp piece count, color-fixing density, and the average mana value weighted against curve.

Win-condition quality. Finisher density and diversity. Penalizes one-trick decks (single removable threat) and rewards redundancy across independent win paths.

Card quality. A coarse proxy using average CMC and the ratio of "dead cards" (expensive creatures with no enter-the-battlefield effect, no evasion, no recursion) to the total non-land count.

Archetype detection

densa_deck.formats.profiles.detect_archetype.

Decks get one of these labels based on role-count + threat-profile heuristics: aggro, midrange, control, combo, stax, aristocrats, spellslinger, tokens, voltron, group_hug, turbo. The logic is a decision tree: "if interaction > 12 and creatures < 15 then control", etc. Every branch has a fallback to midrange so a deck never comes back unclassified.

Mana curve + average mana value

Counted in densa_deck.analysis.static.analyze_deck.

Distribution of non-land cards grouped by mana value (0-7+). Average mana value is the arithmetic mean of all non-land entries weighted by copy count. Hybrid and Phyrexian costs count their converted mana value; X-costs count as 0.

Mana base grade + castability

densa_deck.analysis.advanced.run_advanced_analysis + densa_deck.analysis.castability.

Grade (A-F) is derived from three inputs: pip-demand coverage (does each colored card have enough sources?), untapped-land ratio (how many lands enter tapped?), and fetch/fixing density. A means every colored card you draw is castable on curve; F means frequent color screw.

Castability per card: probability of being able to cast this card on its natural curve turn given the deck's land distribution. Computed by simulating draws for the first N turns (N = card's mana value + 1) and checking whether the required pips + generic mana are available. Low values flag cards whose color demands exceed what the mana base supports (e.g. a triple-green card in a deck with 5 green sources).

Category scores

densa_deck.analysis.static score computation.

Each category (ramp, card draw, removal, wipes, curve) gets a 0-100 score based on how close the deck is to format-appropriate targets:

if actual is in [target_lo, target_hi]: score = 100 if actual < target_lo: score = max(0, 85 - (target_lo - actual) * 10) if actual > target_hi: score = max(0, 85 - (actual - target_hi) * 5)

Targets are format-specific. Commander wants 10-15 ramp, 8-12 draw, 8-12 removal, 2-4 wipes, avg CMC 2.5-3.5. Modern wants 0-4 ramp, 4-8 draw, 6-10 removal, 0-3 wipes, avg CMC 2.0-3.0. Penalties for being over-target are gentler than being under-target because "too much ramp" is rarely as bad as "not enough removal".

Power-level signals

The "+ reasons" and "- reasons" lists are deterministic — each reason corresponds to a specific pattern the tagger detected. Examples: "Strong ramp package" fires when the deck has 10+ ramp pieces in Commander; "Very low interaction" fires when interaction count falls below 60% of the format target. These same reasons feed the AI coach so its explanations are grounded in the same observations you see here.

Hypergeometric probability

densa_deck.probability.hypergeometric.

The hypergeometric calculator answers "given a deck of D cards with K copies of X, what's the chance of drawing at least k copies in the first N cards?" The formula is:

P(X ≥ k) = 1 - sum(i=0..k-1) [ C(K,i) * C(D-K, N-i) / C(D, N) ] where C(n, r) is the binomial coefficient "n choose r".

This is exact, not simulated. A 7-card opening hand from a 99-card Commander deck with 38 lands has P(≥ 3 lands) = ~84%.

Goldfish simulation

densa_deck.goldfish.runner.run_goldfish_batch.

Solo simulation: the deck plays against no opponent for N games (default 1000) and we measure how fast and consistently it executes. For each game:

  1. Mulligan phase — opening hand is kept if it has 2-5 lands AND at least one relevant non-land. Otherwise we mulligan to 6 and repeat, London-style, down to 4.
  2. Turn loop — play a land if possible, cast the highest-impact spell that fits available mana, attack with everything that can swing. No blocker exists (it's a goldfish).
  3. Objective check — Commander goldfishes aim for 40 damage total; 60-card formats aim for 20. We record the turn the deck crosses the threshold, or mark the game a "timeout" if it doesn't by turn 12.

Aggregated outputs: average kill turn (including timeouts as turn 13+ so bad decks' slow games are visible in the mean), objective pass rate, mulligan-down-to-N distribution, and key-card access curves (P(turn T saw ramp/draw/wincon)).

Matchup gauntlet

densa_deck.matchup.gauntlet.run_gauntlet + densa_deck.matchup.simulator.simulate_matchup.

Your deck is simulated against 11 canonical archetype profiles — Aggro, Midrange, Control, Combo, Stax, Aristocrats, Spellslinger, Tokens, Voltron, Group Hug, Turbo. Each archetype is a behavioral model, not a decklist:

  • clock_turns — how fast the opponent threatens lethal unimpeded
  • damage_per_turn — how much they swing for once the clock starts
  • targeted_removal_chance, counterspell_chance, wipe_chance — per-turn disruption probabilities
  • mana_tax, hand_disruption_chance — stax-like pressure
  • resilience — how fast they rebuild after interaction (0-1)

For each archetype we run 200 games (default). Each game plays turns sequentially — opponent interaction first (might strip a card, might wipe), then your turn (play a land, cast best spell, attack). The game ends when either side takes lethal damage or turn 12 is reached. Results roll up into win rate + average turns + damage dealt/taken per archetype.

The weighted meta score multiplies each archetype's win rate by its meta_weight (Aggro = 1.5x, Stax = 0.6x, etc.) so the summary reflects "how well do you do against decks people actually play."

Duel mode

AppApi.duel_decks + densa_deck.matchup.deck_as_opponent.deck_to_profile.

Head-to-head between two of your saved decks. Reuses the gauntlet engine, but instead of pitting your deck against a canonical archetype, we synthesize an ArchetypeProfile for the opponent from their static analysis:

baseline = ARCHETYPES[detected_archetype_of_opponent] clock_turns = 12 - speed * 7 - combo_potential * 1.5 # faster decks have shorter clocks targeted_removal = baseline.targeted_removal * (0.5 + interaction * 1.3) counterspell = baseline.counterspell * (0.5 + interaction * 1.3) hand_disrupt = baseline.hand_disrupt + (0.08 if interaction_count ≥ 10 else 0.04 if ≥ 5 else 0) resilience = 0.2 + mana_efficiency * 0.7 damage_per_turn = baseline.damage_per_turn * (0.6 + speed * 0.8) (all power sub-scores are normalized from 0-10 to 0-1 first)

We then run simulate_matchup twice — once with deck A as the hero and deck B's profile as the opponent, once with the perspectives swapped. Reporting both directions makes the win rate symmetric and reveals cases where the gap is asymmetric (e.g., A beats B 62%, B beats A 58% — still close, but not deterministic).

Games per duel: 100 (default, ~3-5s), 500 (tighter stats, ~15s), or 1000 (benchmark mode, ~30s). Tie threshold for the verdict: win rates within 2 percentage points are labeled "roughly even" to avoid over-interpreting noise.

Caveat: This is not a card-by-card simulation — each deck's concrete cards only influence the opponent via the derived behavioral profile. A true deck-vs-deck sim would play both decks' cards simultaneously; that's slated for a future release.

Recommendations + coach

densa_deck.analysis.static + densa_deck.analysis.advanced for rules; densa_deck.analyst.coach for the LLM layer.

The numeric recommendations ("you're 3 ramp pieces short of the 10-15 target") are purely rule-based — same format targets as the category scores. No LLM involvement, fully deterministic.

The AI Coach layer (Pro feature) reads the same static analysis + a hallucination-proof tag-constrained emission mode: the LLM never types card names directly; it picks from a pre-validated candidate table tagged [c01]..[cNN], and every output card is verified against the deck before rendering. This is how the Coach can recommend concrete cuts/adds without inventing cards.
